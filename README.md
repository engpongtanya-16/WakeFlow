# 🌅 WakeFlow

**An AI-powered morning briefing app that consolidates your calendar, weather, news, and smart assistant into one intelligent dashboard.**

> Built with Python · Plotly Dash · GPT-4o-mini · Google Calendar API  
> Live demo: [wakeflow-production.up.railway.app](https://wakeflow-production.up.railway.app)

---

## 📸 Overview

WakeFlow helps you start every day with full clarity — your schedule, the weather, today's top news, and an AI assistant that can read, create, update, and manage your Google Calendar events through natural conversation.

---

## ✨ Features

### 📅 My Day — Smart Daily Planner
- **Real Google Calendar integration** via OAuth 2.0 — fetches events from all your calendars simultaneously
- **Multi-color Gantt chart** that matches your Google Calendar colors exactly (mapped from API color IDs to hex)
- **Day / Week / Month views** with date picker to navigate to any date
- **AI-generated event tips** — click any event on the Gantt chart to receive 3 personalized preparation tips generated by GPT-4o-mini, using the event title, time, location, and current weather
- **Google Maps embed** — event locations are pulled directly from Google Calendar and displayed as an interactive map with a "Get Route" button linking to Google Maps directions
- **Join Meeting button** — automatically detects Google Meet, Zoom, and Microsoft Teams links from event data and shows a platform-labeled join button
- **Barcelona timezone support** — all times and date calculations use `Europe/Madrid` to avoid UTC mismatches on the Railway server

---

### 🤖 AI Assistant — Conversational Calendar Manager

A full chatbot powered by GPT-4o-mini with **7 tools** that interact with real APIs:

| Tool | What it does |
|------|-------------|
| `get_calendar_events` | Fetches real events from Google Calendar for any date |
| `get_weather` | Gets live weather from OpenWeatherMap |
| `get_news` | Pulls top headlines from NewsAPI by topic |
| `create_calendar_event` | Creates a new event — triggers a review popup |
| `delete_calendar_event` | Deletes an event using fuzzy keyword matching across all calendars |
| `update_calendar_event` | Updates time, location, meeting link, or description of an event |
| `move_calendar_event` | Moves an event from one calendar to another |
| `add_task` | Adds a task to My Tasks — triggers a review popup |

**Key capabilities:**
- **Multi-turn conversation** with full memory of the current session
- **Relative date understanding** — correctly interprets "tomorrow", "this Sunday", "next Monday" using a `[TODAY: Weekday YYYY-MM-DD]` anchor injected into every prompt
- **Tool chaining** — the model decides which tools to call, calls them in sequence, and synthesizes a response using real-time data
- **Fuzzy event matching** — when deleting or updating, uses token-based keyword matching so natural language (e.g. "Louvre Museum") finds "Lourve Museum" in the calendar
- **Calendar picker popup** — when creating an event, a modal pops up for the user to review and edit the title, date, time, description, location, meeting link, color, and target calendar before confirming
- **Task picker popup** — when adding a task, a modal lets the user edit the text, category, and due date before confirming
- **👍 / 👎 voting** on AI responses, persisted to `feedback.json` for iterative improvement
- **Tool usage analytics** — tracks how often each tool is called, displayed as a live bar chart in User Info

---

### 📋 My Planner — Smart Event Import

- **Upload a PDF or image** of a schedule, syllabus, or timetable
- **AI extracts structured event data** (title, date, time, description) using GPT-4o-mini with vision support for images
- **Review cards** for each extracted event — editable before adding:
  - Toggle between **Calendar Event** or **Task**
  - Edit date, start time, end time per card
  - Choose **target calendar** per card (each event can go to a different calendar)
  - Delete unwanted items with ✕
  - Button label updates live as you delete cards
- Add all remaining items in one click

#### ✅ My Tasks
- Persistent to-do list stored in **browser localStorage** (survives refresh and redeployment)
- Categories: To-Do, Assignment, Work, Personal
- Due-date badges with color warnings (overdue = red, due today = orange, due soon = yellow)
- Tasks can be added manually, imported from uploaded files, or created by the AI chatbot

---

### 🌤️ Weather — Live Forecast
- Real-time weather data from **OpenWeatherMap API**
- Displays temperature, feels like, humidity, and weather condition
- **Smart outfit advice** based on temperature range
- **Visual bar chart** with dual-axis display (temperature + humidity)
- **City search bar** in the Weather tab — search any city independently without changing the User Info default

---

### 📰 News — Personalized Headlines
- Live headlines from **NewsAPI** with multi-endpoint fallback
- **Topic filter chips** — toggle Tech, Finance, World, Business, Science, Sports
- **Color-coded cards** by topic (each topic has a distinct accent color and background)
- Article topic is auto-detected by keyword matching against selected topics
- Clicking "Read →" opens the full article in a new tab
- Topic preferences sync with User Info settings

---

### 📧 Daily Email — Automated Morning Briefing
- **Scheduled HTML email** sent daily at a user-configured time
- Powered by **APScheduler** cron job with `Europe/Madrid` timezone (so "7:00" means 7:00 Barcelona time)
- Email includes: personalized greeting, today's calendar events, current weather, and top 5 news headlines
- Sent via **Brevo API** (transactional email)
- UTF-8 normalized to prevent encoding failures from LLM-generated characters

---

### 👤 User Info — Settings & Preferences
- Set your **default city** for weather, AI assistant, and daily email
- Select **news topics** that propagate across News tab, AI assistant, and email
- Configure daily email: recipient, name, send time
- **Tool Usage Analytics** — bar chart showing how often each AI tool is called
- **AI Response Voting stats** — running thumbs up/down count

## 🚀 Setup & Running Locally

### 1. Clone the repo
```bash
git clone https://github.com/engpongtanya-16/WakeFlow.git
cd WakeFlow
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
pip install pdfplumber  # for PDF event extraction
```

### 3. Create `.env` file
```env
OPENAI_API_KEY=your_openai_key
WEATHER_API_KEY=your_openweathermap_key
NEWS_API_KEY=your_newsapi_key
BREVO_API_KEY=your_brevo_key
REDIRECT_URI=http://localhost:8050/oauth2callback
GOOGLE_CREDENTIALS={"web": {...}}   # paste your credentials.json content
```

### 4. Run the app
```bash
python app.py
```
Open [http://localhost:8050](http://localhost:8050)

## 🔐 Google OAuth Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Create a project → Enable **Google Calendar API**
3. Create **OAuth 2.0 credentials** (Web application type)
4. Add authorized redirect URI: `http://localhost:8050/oauth2callback`
5. Download `credentials.json` and place in the project root
6. On first run, go to **Settings → Google Calendar → Connect**
7. Complete OAuth flow — token saved to `google_token.json`

> ⚠️ The app is currently in **Google OAuth testing mode**. To access Google Calendar features, your Gmail address must be added as a Test User in Google Cloud Console.

## 📦 Deployment (Railway)

1. Push code to GitHub (ensure `.env`, `google_token.json`, `credentials.json` are in `.gitignore`)
2. Create a new Railway project linked to your GitHub repo
3. Add environment variables in Railway dashboard (same as `.env`)
4. Add `GOOGLE_CREDENTIALS` variable with the full JSON content of `credentials.json`
5. Add `REDIRECT_URI` pointing to your Railway domain: `https://your-app.up.railway.app/oauth2callback`
6. Deploy — Railway auto-detects Python and runs `python app.py`

---

## ⚠️ Known Limitations

- **Google OAuth testing mode**: Only pre-authorized Gmail accounts can connect Google Calendar. Up to 100 test users can be added via Google Cloud Console.
- **Task persistence**: Tasks are stored in browser localStorage — they persist across refreshes on the same browser but not across devices.
- **NewsAPI free tier**: Limited to 100 requests/day and articles older than 1 month are restricted.
- **APScheduler on Railway**: The scheduler resets on each deployment — users need to re-save their email schedule after a new deploy.
